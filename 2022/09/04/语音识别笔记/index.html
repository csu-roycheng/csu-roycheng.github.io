语音识别笔记

Created2022-09-04|Updated2022-09-04|语音
|Post View:

模型介绍

LAS

Listen：编码 acoustic feature，encoder 可以是 RNN、CNN、CNN + RNN、self-attention 等等。

Attention：Bahdanau、Luong attention 等等。

Spell：一个 decoder cell 输出一个概率分布。

训练时使用 Teacher Forcing 方法。

问题：不能实时语音识别，因为 encoder 编码时需要看到完整的语音。

CTC

  • 输入一个 acoustic feature，输出一个高维特征，并通过一个简单的网络 classifier 输出 token distribution，实现流式语音识别；
  • encoder 是单向 RNN；
  • 由于 acoustic feature 比输出的 token sequence 长很多，因此需要做输入和输出长度的 alignment，具体来说是在输出的 token 中加上一个特殊符号 $\phi$，表示空输出；
  • 最后输出序列需要将重复字符合并，然后删除 $\phi$

问题 1：训练数据并没有对齐的 ground truth，因此要枚举出所有 alignment，作为训练集。

问题 2：CTC 每一步输出是独立的，没有看之前输出的内容，因此容易产生错误的输出结果（结巴）。

RNN-T

  • 使 decoder 满足输入一个高维特征能够输出多个 token，输出 $\phi$ 时再输入下一个特征
  • 训练数据需要有与 acoustic feature 长度个 $\phi$，这又会产生训练数据对齐的问题，枚举所有的 alignment

实际上，RNN 是另外训练的，以解决结巴问题。

Neural Transducer

  • 相比 CTC、RNN-T 一次只考虑一个高维特征，Neural Transducer 一次考虑多个高维特征
  • 当输出 $\phi$ 时，输入下一个 chunk
  • 问题：一次输入的特征是固定的
  • MoCha：动态选择输入的特征数量

计算对齐的概率

针对 RNN-T

$p_{a,b}$ 表示已经读到第 a 个 feature，并已经产生了 b 个 token，此时输出的概率分布。

下图表示：虽然有很多条路径到 $p_{4,2}$，但是 $p_{4,2}$ 的值是固定的。

因为 $p_{4,2}$ 的计算只依赖于输出的 token 和输入的 feature，与 $\phi$ 无关。

设 $\alpha_{i,j}$ 为已经读到第 i 个 acoustic feature 且生成了第 j 个 token 时，所有 alignment 概率之和。例如 $\alpha_{4,2}=\alpha_{4,1}p_{4,1}(a)+\alpha_{3,2}p_{3,2}(\phi)$。那么右下角的 $\alpha_{i,j}$ 即为所有 alignment 概率之和，即 $\alpha_{i,j}=P_\theta(Y|X)$。

由于 $\alpha_{i,j}$ 可以由上面一格和左边一格计算得到，因此可以使用动态规划的思想进行计算。

训练

在计算 alignment 的时候，我们需要根据训练好的 RNN 才能计算出来 $P_\theta(Y|X)$，θ 为 RNN 的参数。

目标函数：$\theta^*=arg\max_\theta\log P_\theta(\hat Y|X)$。

如何计算 $\sum_{p_{4,1}(a)}P(h|X)$？引入 $\beta_{i,j}$，表示读到了第 i 个 acoustic feature 且生成了第 j 个 token 的 alignment，到结束节点的分数总和。可知 $\beta_{i,j}$ 由右边和下面两个状态来计算。

知道 $\alpha$ 和 $\beta$ 之后，$\sum_{p_{4,1}(a)}P(h|X)=\alpha_{4,1}p_{4,1}(a)\beta_{4,2}$：

Inference

用分数最大的 alignment 来代替所有 alignment 的分数之和。

可以用 beam search 来获得更准确的结果。

总结

  • LAS 和 RNN-T 的 decoder 会考虑之前输出的内容，而 CTC 不考虑
  • CTC 和 RNN-T 需要做对齐
  • CTC 和 RNN-T 的训练更加麻烦
  • CTC 和 RNN-T 可以做到实时语音识别
