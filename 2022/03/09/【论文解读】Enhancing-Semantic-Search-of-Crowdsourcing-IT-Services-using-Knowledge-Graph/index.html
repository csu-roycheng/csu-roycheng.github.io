Enhancing Semantic Search of Crowdsourcing IT Services using Knowledge Graph（2019 SEKE）

简介

目前，几乎所有众包平台都按照图 1 所示的基本搜索流程提供搜索功能。服务数据用于创建倒排索引。然后 Elasticsearch 在查询和倒排索引之间执行文本匹配。该方法采用纯文本匹配技术，这意味着用户必须精确地描述他们的需求，否则，它无法识别用户的潜在意图。

与通用搜索不同的是，软件众包服务的搜索是垂直领域搜索，其搜索意图范围更窄，领域性更强，需要领域知识的语义补充。

同时，大多数软件众包平台的服务提供者通常使用图片信息来描述服务内容，可用于语义特征匹配的文本描述较少。

针对以上问题，本文利用所构建的软件众包服务知识图谱，将领域知识融合到搜索模型中，利用语义扩展和主题模型提取出搜索语句和服务之间隐含的语义匹配特征，使用学习排序模型，得到按照相关度排序的搜索结果。

方法介绍

如图 2 所示。我们的方法利用 IT 服务知识图（ITServiceKG）来增强查询和服务理解，然后在基本搜索过程中添加两个额外步骤，以提供精确的 IT 服务搜索：语义扩展和服务重排序。

知识图谱构建

在软件众包平台上，用户根据自己的需求搜索目标软件服务。而服务商家根据自己提供的技术栈给出详细的服务描述。因此，作者将知识图谱分为三大部分：服务类目图谱、服务技能图谱和服务实例图谱。

在 ITServiceKG 中，每个实体（即概念实体或实例实体）都有以下属性：名称、类型和描述；关系包含：同义词、共现词、上位词、下位词等等。

服务类目图谱

服务类目图谱从用户需求角度，给出软件服务的不同类目信息。该部分记录服务类目的层次结构。

一共分为三层：

  • 第一层包含各种类型的软件服务，包括软件开发、软件测试、架构设计、DevOps 部署等；
  • 第二层收集特定 IT 服务的子类型。以 “软件开发” 为例。其子服务包括 “web 开发”、“应用程序开发”、“嵌入式系统开发” 等软件服务；
  • 第三层是面向领域的 IT 服务。

服务技能图谱

服务技能图谱从服务商家提供的技术出发，描述了服务的技术栈信息。它包括 11 大技术栈：编程语言、数据库、软件框架、算法、操作系统、软件包、中间件、存储技术、运维技术、测试技术、开发工具。

每项技术都有一个多层结构。以 “数据库” 为例，它可以分为 “关系数据库” 和 “非关系数据库”，MySQL是 “关系数据库” 的一个实例。

服务实例图谱

服务实例图谱在 ITServiceKG 中负责连接服务类目图谱和服务技能图谱。

如图 3 所示，图中右上部分为服务类目图谱部分，左边和右下部分为服务技能图谱部分，中间由软件服务的各个服务实例实体（如图中的 No. 1365 服务）与服务技能图谱中概念实体的各个实例实体（如编程语言实体“Java”）之间的多种关系相连。

从例子中可以看到，No. 1365 使用的框架是 Structs 和 Hibernate，使用的数据库是 Redis 和 MongoDB。

语义扩展

作者认为语义扩展存在改进空间：

  • 在以往的研究中，查询扩展词来自同义词和共现词，忽略了上下位词的关系；
  • 大多数研究只关注查询扩展，而忽略了文档扩展。在 IT 服务众包平台中，没有足够的服务文本描述，这影响了语义搜索的性能。

因此，用户查询和服务信息的扩展是必不可少的，作者基于 ITServiceKG 使用以下四个方法进行语义扩展：

  1. 同义词和共现词
  2. 上下位词
  3. 使用实体描述作为伪查询语句
  4. 利用实体的描述对服务信息做语义扩展

下面以搜索语句“餐饮管理系统移动应用开发”为例，说明扩展的方法。

该搜索语句包含两个实体：“餐饮管理系统” 和 “移动应用开发”。

  • 对 “餐饮管理系统” 进行同义词扩展，同义词如 “点餐系统”，“点菜系统”，“点餐平台”等；
  • 对 “移动应用开发” 做下义词扩展，如 “app 开发”，“微信小程序开发”，“H5 开发” 等。

“餐饮管理系统”的实体描述为：“餐饮管理系统是服务于餐馆的日常管理的，是为了满足餐饮业发展，科学管理，餐馆管理、调高效率的管理系统。”

  • 可以将中实体的描述当作为伪搜索语句作为原搜索语句的补充。

下面以服务 “餐饮点菜系统-微信小程序” 为例，说明对服务信息进行语义扩展的方法。

该服务的描述信息中包含一些重要的实体，如 “微信小程序”，“餐饮外卖”，“生鲜配送”，“生活服务” 等等。将这些实体的描述用于服务信息的扩展补充，起到对服务的解释说明作用。

服务重排序

通过语义扩展之后，首先对搜索语句和服务信息进行一个初步的语义匹配。

作者在 ElasticSearch 中对语义扩展后的服务描述信息建立了倒排索引，并根据语义扩展后的查询语句，将所有相关的服务尽可能的召回，也就是做一个初步的筛选。

然后，作者应用机器学习模型对搜索结果列表进行重排序。模型的输入是查询语句和所有要排序的 IT 服务，输出是 Top N 的服务。

作者采用了 GBDT 作为 LTR 模型，并为该模型设计了三个相关度特征：

  1. 单词重叠程度：如果查询和服务的名称和描述有更多的公共词，它们就更相关。

  2. 嵌入向量的相似性：通过 TD-IDF 将查询、服务名称和描述嵌入到向量中。查询和服务之间向量的余弦相似度表示相关性。

  3. 查询-服务的相关性：在 IT 服务众包平台中，在搜索日志中将服务点击事件记录为 $<query,service,timestamp>$ 三元组，点击次数越多，相关性越高。

总结

搜索流程为：

  1. 通过知识图谱扩展查询语句和服务信息
  2. 采用文本匹配的方式获得初步的召回
  3. 根据步骤 2 中的查询-服务对，采用 LTR 模型对 2 中的结果重新排序
